# Spotify Landing Page - Prompt Creativo

## Role: 
Act as a senior frontend developer specialized in UI/UX design and user experience, with expertise in high-performance, high-conversion landing pages.

## Task: 
Build a complete, publication-ready landing page promoting Spotify as a music streaming application. The page must be visually striking, modern, and optimized to convert visitors into users.

---

## 🎯 PRIMARY GOAL
Design and develop a Spotify landing page that communicates its value proposition, generates desire to download/subscribe, and delivers a smooth, accessible, and memorable user experience.

---

## 📐 REQUIRED SECTIONS & STRUCTURE
Build the page with the following sections in order:

1. **Hero Section** — Powerful headline, subheadline, primary CTA ("Listen Free" / "Try Premium"), device image or animation showing the app in use.

2. **Value Proposition** — 3–4 key features with icons: millions of songs, podcasts, ad-free listening, offline mode.

3. **Plan Comparison** — Free vs Premium (pricing table or cards).

4. **Social Proof** — User testimonials, active listener count, star ratings.

5. **Visual Demo** — Animated app player mockup or UI screenshot carousel.

6. **Final CTA** — Closing section with urgency-driven call to action and offer (e.g., "3 months free").

7. **Footer** — Legal links, social media, language selector.

---

## 🎨 UI/UX DESIGN REQUIREMENTS

- **Color palette**: Deep black `#121212`, Spotify green `#1DB954`, white `#FFFFFF`, and dark grays.
- **Typography**: Use "Circular" (or alternatives like "DM Sans" / "Figtree") for headlines; clean, readable font for body text.
- **Aesthetic**: Elegant dark mode, minimalist yet energetic. High contrast. Premium feel.
- **Animations**: Button micro-interactions, smooth section entrances on scroll (fade-in + slide), animated equalizer bars as a decorative element.
- **Responsive design**: Mobile-first, optimized for mobile, tablet, and desktop.
- **Accessibility**: Minimum WCAG AA contrast ratio, ARIA roles, full keyboard navigation support.

---

## ⚙️ TECHNICAL TASKS TO COMPLETE
Ensure the code includes or addresses:

- ✅ Semantic HTML5 (header, main, section, footer with correct roles)
- ✅ CSS with custom properties (--color-green, --color-bg, etc.)
- ✅ CSS and/or JS scroll-reveal animations
- ✅ CTA buttons with hover/focus/active states
- ✅ Email capture form with basic validation
- ✅ SEO meta tags: title, description, Open Graph, Twitter Card
- ✅ Favicon and basic PWA manifest
- ✅ Images with descriptive alt attributes and lazy loading
- ✅ Efficiently loaded Google Fonts or self-hosted fonts
- ✅ Performance: optimized images, critical CSS inlined, deferred JS
- ✅ Single deployable HTML file (or self-contained React component)

---

## ✍️ COPY & CONTENT
Write all page copy yourself:

- Benefit-driven, impactful headlines
- Microcopy for buttons, labels, and tooltips
- Feature descriptions in 2 lines maximum each
- Minimal legal text in footer (terms, privacy)

---

## 📦 EXPECTED DELIVERABLE
A single HTML file (or .jsx if using React) that is fully functional and:

- Opens in any browser without critical external dependencies
- Has all sections implemented with real content — no lorem ipsum or placeholders
- Looks and feels distinctly like Spotify, not a generic template
- Includes inline code comments where relevant for future maintenance
